# Feature Research（策略 + 競品 + 用戶 + 增長）

## 標準指令區塊

知識實時性：涉及技術方案時，先聯網搜索確認最新資訊。
引用規範：所有來源用 [來源N](URL) 封裝，嚴禁裸露 URL。
誠實原則：找不到精準來源時直說，嚴禁資訊幻覺。
務實原則：只解當前真實問題，最簡方案，拒絕過度設計。
探針提問：請求模糊時，問一個最關鍵的澄清問題，確認後再執行。

## §1 產品策略

### OKR 框架

```
Objective（目標）：[用一句話說明你想達成什麼，例：成為目標市場的首選 App]

Key Results（關鍵結果）：
KR1：[時間段] 內達到 [MAU 目標]
KR2：App Store 評分 ≥ [目標]
KR3：D30 留存率 ≥ [目標]
KR4：[主要收入指標] 達到 [目標]

季度 Review：KR 達成率 >70% 繼續 / <50% 調整策略
```

### 路線圖框架

```
Phase 1（0-3月）：核心 MVP
  目標：解決核心用戶問題，可以上架
  功能：[你的核心功能，≤5個]

Phase 2（3-6月）：增用戶
  目標：建立用戶習慣，DAU 穩定增長
  功能：[留存機制 + 增長功能]

Phase 3（6-12月）：商業化
  目標：收入可持續
  功能：[廣告 / 訂閱 / 高級功能]

Phase 4（12月+）：擴張
  目標：進入新市場或新用戶群
  功能：[新語言 / 新平台 / 社交功能]
```

### 功能取捨：3個問題

```
每個功能都問：

1. 服務哪個 Job-to-be-Done？
   → 找不到用戶真實需求就不做

2. 增加還是降低留存率？
   → 優先做增留存的功能

3. 對主要收入指標有正向影響？
   → 增加 DAU / Session 時長 / 轉化率優先
```

### RICE 優先級評分

```
RICE = (Reach × Impact × Confidence) / Effort

Reach（觸及用戶比例）：1-10
Impact（每用戶影響）：0.25 / 0.5 / 1 / 2 / 3
Confidence（確定程度）：50% / 80% / 100%
Effort（人天，越小越好）

> 5 本Sprint做 | 2-5 下Sprint | < 2 Backlog
```

### 差異化定位

```
市場空白（競品沒做好的地方）：
• [競品A 的主要差評] → 你的解法
• [競品B 的主要差評] → 你的解法

你的差異化：
「[你的App] = [強項A] + [強項B] + [目標用戶優化]」
```

### 1-star 差評挖掘（每月一次）

```
App Store / Play Store → 按「最差評分」排序 → 讀前50條差評
→ 每一條差評都是你的設計機會

行動：把每條差評變成你的設計決策
例：「廣告太多」→ 限制廣告頻率
   「功能 X 不好用」→ 你的 UX 改善點
```

## §3 用戶研究

### Jobs-to-be-Done 框架

```
用戶「僱用」你的 App 做什麼工作？

找出你的主要 Jobs（按使用頻率排序）：
Job 1：[場景描述]（預計占 X% 使用場景）
Job 2：[場景描述]
Job 3：[場景描述]

最重要的 Job → 設計決策
例：如果 Job 1 = 「在工作中快速查東西」
→ 速度和搜索質量比設計更重要
→ Onboarding 要讓用戶 < 30 秒完成第一個核心動作
```

### Aha Moment 定義

```
你的 App Aha Moment = 用戶第一次真正體驗到核心價值的那一刻

例：
• 工具類 App → 第一次用功能解決了真實問題
• 社交類 App → 第一次與真實朋友互動
• 學習類 App → 第一次完成一個學習里程碑

定義你的 Aha Moment：[填入]
→ Onboarding 設計的核心目標是讓用戶盡快到達這個時刻
```

### 用戶訪談問題框架

```
✅ 問過去行為（不問假設）：
• 上週你最後一次用[同類App]是什麼情況？
• 你做完之後做了什麼？
• 有沒有讓你很煩躁的地方？
• 你試過其他 App 嗎？為什麼換（或不換）？
• 如果有一個功能消失了，你最想保留哪個？
• 你會推薦這個 App 給朋友嗎？為什麼？

❌ 不要問：
• 你想要 X 功能嗎？（會說「想要」）
• 你會用 Y 嗎？（假設性問題不可靠）
```

### 用戶分群框架

```
根據你的 App，識別 2-3 個核心用戶群：

A 群：[用戶描述]
  需求：[他們主要用 App 做什麼]
  特點：[使用頻率、付費意願、使用場景]
  設計重點：[對這群用戶最重要的功能/體驗]

B 群：[用戶描述]
  需求：...
  特點：...
  設計重點：...
```

## §4 增長策略

### 廣告收入公式（廣告變現 App）

```
廣告收入 = DAU × Session次數 × Session時長 × 廣告曝光/分鐘 × eCPM / 1000

優先優化順序：DAU → Session次數 → Session時長
```

### 北極星指標

```
定義你的北極星指標（最能代表用戶獲得核心價值的指標）：

[你的北極星] = [用戶完成核心動作的次數/天]

例：
• 工具類 → 每日成功使用核心功能次數
• 社交類 → 每日有效互動次數
• 學習類 → 每日學習完成次數
• 電商類 → 每日完成購物流程次數
```

### D1/D7/D30 留存目標與診斷

```
行業基準（Mobile App 平均）：
D1 > 25-40%  D7 > 10-20%  D30 > 5-10%

根據你的 App 類型設定目標，然後診斷：

D1差（< 行業基準 × 0.7）→ Onboarding問題，Aha Moment沒發生
  改善：降低首次使用門檻、讓用戶更快到達Aha Moment

D1好但D7差 → 沒有理由回來
  改善：習慣機制（通知、Streak）、每日新內容/功能

D7好但D30差 → 新鮮感消退
  改善：個人化、社交功能、進度追蹤
```

### Hook Model（讓用戶每天回來）

```
觸發 → 行動 → 獎勵 → 投入

為你的 App 設計：

觸發（外部）：
• [Push通知策略，例：進度提醒、新內容]
• [Email/其他渠道觸發]

行動（最小化摩擦）：
• 通知點擊直接進核心功能（不是主頁）
• [你的核心動作，目標：≤2步到達]

獎勵（可變的）：
• [用戶完成核心動作後的即時回饋]
• [進度/成就系統]

投入（越用越難離開）：
• [用戶創造的個人數據：歷史、進度、設定]
• [累積的個人化數據]
```

### Onboarding 漏斗

```
每增加一步流失10-15%，目標≤5步到達Aha Moment：

✅ 直接進入核心功能（不要立即要求登入）
✅ 第一步就讓用戶完成一個有意義的動作
✅ 通知權限在用戶看到價值後才請求（Session 3+）
✅ 登入/註冊在用戶準備「保存」什麼之前不要問
```

### 病毒係數 K-factor

```
目標 K > 0.3（每用戶帶來0.3個新用戶）

分享機會（根據你的 App 設計）：
• [用戶成就 / 進度分享卡]
• [邀請朋友共同使用]
• [分享有趣內容]
```
